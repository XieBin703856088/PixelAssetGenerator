# 32×32 像素艺术算法规范 V2

## 目标

节点输出应当像“直接在 32×32 画布上绘制的像素画”，而不是缩小后的连续程序纹理。

核心规则：

- 单个素材通常使用 4–12 色，技术贴图除外。
- 明暗采用离散色阶，不使用连续渐变或亚像素抗锯齿。
- 纹理细节必须形成可读的 2–8 像素簇，避免随机单像素噪点。
- 轮廓使用 4 邻域或菱形形态学运算，不使用 Sobel 软边作为最终描边。
- 材质变化以大块面、中等结构和少量微细节三个尺度组织。
- 可平铺节点的邻域、像素簇清理和形态学操作必须环绕采样。
- SpriteSheet、切片、通道、法线和距离场等数据节点禁止自动改色。

## 两层运行结构

1. 原生像素算法：高频节点直接生成离散轮廓、色阶和像素簇。
2. 兼容风格化层：旧视觉脚本输出由 `PixelArtKernel` 进行自适应限制色和孤立簇清理。

自动风格化覆盖：

- Generate
- Material
- Noise
- 视觉型 Pattern
- 视觉型 ImageProcess

保持原样的技术型输出：

- SpriteSheet / SpriteSlice / NineSlice / TileCombine
- Channel / NormalMap / DistanceField / Mask 等数据节点
- Logic / Tool / Physics / Particle 数据流

## 当前原生 V2 节点

- Grass：离散草叶、土壤斑块、丛生结构、5 色阶。
- Rock：透明 Sprite 轮廓、方向光分面、粗糙块面。
- Cobblestone：环绕 Voronoi 石块、离散缝隙、高光/阴影边。
- WaterFlow：可平铺方向波带、4 色水体和像素泡沫。
- LavaFlow：环绕裂隙、熔壳/熔岩/高温核心分层。
- Pixelate：块级采样、限制色与可选块面描边。
- ColorQuantize：感知自适应调色板，32 色上限。
- Outline：离散内/外形态学描边与双阶光晕。
- Lighting：法线/高度输入、四级明暗和限制色输出。
- PixelArtStyle：通用限制色、抖色、透明度和像素簇整理。

## 验收指标

- 32×32 默认输出颜色数不超过节点规定的调色板上限。
- 默认材质不得只输出单色。
- Sprite 类节点同时具有透明区和有效轮廓区。
- 可平铺算法的采样与清理使用环绕坐标。
- 1 像素孤立色块应被合并，显式粒子/星点节点除外。
- 节点输出对相同参数和种子完全确定。
