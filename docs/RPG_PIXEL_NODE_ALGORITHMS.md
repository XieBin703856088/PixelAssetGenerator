# RPG 像素节点算法规范

## 两类绘制内核

- **Sprite 节点**：树木、灌木、蘑菇。使用二值遮罩、硬边膨胀描边、分层遮挡和方向光色块，透明边缘只允许 `0/1` Alpha。
- **Tile 节点**：草地、鹅卵石、布料、皮革。使用可回绕的调色板索引画布，所有跨边界的草叶、接缝和褶皱都写入对侧，保证平铺连续。

## 32×32 / 64×64 规则

1. 轮廓和关键结构先按画布比例定位，再吸附到整数像素。
2. 32 像素强调剪影和主色块；64 像素增加结构细节，而不是简单双倍放大。
3. 默认结果控制在 3–10 个颜色以内，不使用逐像素连续渐变。
4. 明暗以成组像素表达：左上高光、右下阴影、接触部位遮挡暗色。
5. 固定种子必须得到完全相同的像素数据。

## 回归检查

`tools/GraphCoreSmoke` 会验证：处理器注册、有限调色板、Sprite 二值 Alpha、固定种子确定性、Tile 全覆盖和边界接缝能量。设置 `PIXEL_SPRITE_QA_PATH` 后还会导出 32/64 双尺寸视觉对照图。
