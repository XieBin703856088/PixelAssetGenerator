# AI 像素素材生成节点

## 目标

`AiImageGen` 是显式触发的本地生成节点，用于制作 RPG 精灵、角色、图标、地块和小场景。输出尺寸始终读取项目的全局图块大小。编辑提示词或刷新画布不会自动执行扩散模型；只有属性面板中的“开始生成 / 重新生成”会创建新请求。

## 默认模型

- 语义底模：Stable Diffusion v1.5 pruned EMA-only Q8_0 GGUF
- 底模文件：`stable-diffusion-v1-5-pruned-emaonly-Q8_0.gguf`
- 底模大小：1,763,578,176 字节（约 1.64 GiB）
- 底模 SHA-256：`d0555243938c62faeefb4ac93f6c7a053ad373a4290c5256bce229aeb193bf94`
- 底模许可：CreativeML Open RAIL-M
- 底模页面：https://huggingface.co/second-state/stable-diffusion-v1-5-GGUF
- 像素适配器：`VaroDZAKY/Varo_pixel_Art`
- 适配器文件：`Varo_pixel_Art.safetensors`（12,795,512 字节）
- 适配器 SHA-256：`3c50888c1b0e9a6905052abdc5ca997a2ad7f5936c91582f4ebb96c18a537350`
- 适配器许可：Apache-2.0
- 适配器页面：https://huggingface.co/VaroDZAKY/Varo_pixel_Art

模型通过 Git LFS 存放在项目仓库的 `像素素材生成器/Models/Bundled`，构建和发布时复制到软件的 `Models/Bundled` 目录。最终用户无需联网或自行下载模型。图像模型的固定大小和 SHA-256 标记会在启动时检查；文件缺失时软件只提示重新安装完整离线版本，不会在运行期发起下载。

中文提示词由内置的 `Xenova/opus-mt-zh-en` INT8 ONNX 模型在 CPU 上先翻译为英文语义，避免占用 A 卡显存。翻译模型缺失或推理失败时自动回退到 RPG 像素素材领域词典。

## 显卡后端

运行时使用 `StableDiffusion.NET` / `stable-diffusion.cpp`，按优先级选择：

1. CUDA 12：NVIDIA，系统检测到可用 CUDA 12 时启用。
2. Vulkan：AMD、Intel、NVIDIA；Windows 上检测到 Vulkan loader 时启用。
3. CPU：没有可用 GPU 后端时回退。

属性面板会显示本次进程选择的实际后端。Vulkan 路径使 Windows A 卡不需要安装 ROCm。

## 生成流程

1. 软件直接检查并读取随安装包分发的图像模型和中文翻译模型，无模型下载步骤。
2. 节点收集提示词、素材类型、RPG 风格限制、构图视角、全局图块大小、参考图、蒙版、采样参数和请求版本。
3. 无参考图时执行文生图；连接参考图时执行图生图；连接蒙版时传入重绘蒙版。
4. 生成前先把对象语义交给通用 SD1.5 底模，再以中等权重叠加像素 LoRA；不再依赖旧 checkpoint 的 `pixelsprite/16bitscene` 触发词。风格、视角和对象结构约束随后自动拼接。建筑会补充完整体块、入口、屋顶、窗格和楼层语义，并排除自动售货机、柜体和破碎立面等常见误生成。
5. 石块、石头和岩石会统一为单个天然灰色岩石语义，并排除彩虹色、霓虹、晶体、矿洞、糖果、史莱姆和抽象色块等常见误生成。
6. 对模型难以稳定构图的建筑，在没有参考图时自动生成与所选视角匹配的低分辨率楼体骨架，并通过低强度图生图保留楼体、侧墙、屋顶、窗格和入口；用户参考图始终优先。
7. 模型在 512×512 生成后先逐级降到 128/256 结构层，执行边缘保持和区域归并，再通过局部主色与结构中心采样生成 32/64 像素逻辑网格，不再直接从 512 做一次面积平均。
8. 对最终网格执行风格限定的调色板上限、微纹理抑制、孤立像素清理、像素簇重建、可选弱抖动、蒙版 Alpha 和一像素轮廓。
9. 结果按画布节点实例缓存。后续图计算只复制缓存，不重复推理。

## RPG 风格与视角

- 风格限制：经典 32×32、精细 64×64、复古 16 位 JRPG、暗黑奇幻、清新生活、科幻战术。
- 构图视角：自动匹配、45° 等距斜视、正面俯视、正俯视、正面平视、侧面。
- 风格选项不仅添加提示词，还限制调色板数量、抖动强度、微纹理和平滑/像素簇清理次数。
- 无缝地块始终强制正俯视，避免与透视视角产生冲突。

## 参数建议

- 物件精灵/图标：背景处理“自动”，轮廓开启；经典 32×32 风格最多使用 12 色。
- 角色：建议将全局图块大小设为 64×64，使用 16–24 色，参考图强度 0.45–0.65。
- 地块/场景：背景处理“保留背景”，轮廓关闭，16–32 色。
- 参考图重绘强度越高，越偏离原图；越低，越保留原构图。
- 中文提示词会先经过内置小型翻译模型；风格、视角和对象结构限制使用稳定的英文固定词自动补充，RPG 领域词典继续作为翻译兜底。

## 发布说明

项目当前包含 CUDA 12、Vulkan、CPU 三套 Windows 原生后端，以及约 1.78 GiB 的图像生成组合和约 108 MiB 的中文翻译模型，因此完整 Release 输出会明显增大。大模型文件必须通过 Git LFS 克隆；可在发布阶段拆分为全兼容包与厂商精简包，但完整离线包必须保留 `Models/Bundled`。
